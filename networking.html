Common ports cheat sheet

Ports aren't the main issue by themselves—but knowing what should be open and different rules helps spot mistakes fast.
(In a lab, try to default-deny inbound and choose to allow only what you need whether it be locally or other.)

Port / Proto Service What it's used for Homelab note
22/TCP SSH Remote shell, file copy (SCP/SFTP) Use keys + disable password auth if possible
53/UDP,TCP DNS Name resolution Internal DNS (Pi-hole/Unbound) is a huge quality-of-life upgrade
67-68/UDP DHCP IP assignment Keep DHCP authoritative per VLAN/subnet
80/TCP HTTP Web traffic (plaintext) Usually redirect to HTTPS
443/TCP HTTPS Encrypted web traffic Reverse proxy + TLS termination lives here
123/UDP NTP Time sync Time drift breaks TLS, logs, and authentication
25/TCP SMTP Mail transfer Most labs use relay services; avoid exposing your own mail server
587/TCP SMTP Submission Authenticated sending Prefer this over raw 25 for clients
445/TCP SMB Windows file sharing Keep internal-only; segment untrusted devices
2049/TCP NFS Unix/Linux file sharing Great for VM/container storage; lock to subnets
3000/TCP App ports Common dev/app default (varies) Put apps behind a proxy instead of exposing random high ports
3306/TCP MySQL/MariaDB Database Don't expose DBs directly; allow only app subnets
5432/TCP PostgreSQL Database Same idea: internal access only
8080/TCP Alt HTTP Web apps / admin UIs Proxy it to 443 with a hostname
Quick mental model:
TCP is “reliable stream”,
UDP is “fast packets”.
DNS uses both: UDP for most queries, TCP for larger answers/zone transfers.

Protocols you'll see everywhere

  • ARP: “Who has this IP?” on a local network.
  • ICMP: Ping/traceroute and network diagnostics.
  • DHCP: Hands out IP, gateway, DNS to clients.
  • DNS: Names ↔ IPs (and a lot more, like TXT records).
  • TLS: Encryption + identity for HTTPS and other services.

Basic homelab network pattern

If you only do one thing: create separate networks (VLANs/subnets) and specify trust levels.

  • LAN: your computers/phones
  • LAB: servers/VMs/containers
  • IoT: TVs, cameras, smart devices (limited access)
  • Guest: internet-only

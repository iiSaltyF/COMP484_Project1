My personal workflow (containers)

  • Use Compose for single-host apps (fast, readable, easy to back up).
  • Put apps behind my personal reverse proxy and cloudflared to use hostnames + HTTPS.
  • Persist data in volumes, and back up volumes/configs explicitly.
  • Keep an “apps” repo with one folder per service: compose.yaml + notes.
Containers are great for apps.
For “pet servers” (AD, NAS, routers), VMs often behave better.
I only really use VMs as an alt

Docker alternatives (when they shine)

Podman
Docker-like CLI, rootless-friendly, good for tighter security models.
LXC/LXD
System containers (feel like lightweight VMs). Great for “almost a VM” workloads.
VMs
Strong isolation, easiest for OS-level services and “appliance” style systems.
Kubernetes
Powerful orchestration, but more moving parts—best when you truly need multi-node or scaling.

Picking the right tool

  • Docker Compose if: Single host + a few apps?
  • VM if You Need: OS control/kernel modules and heavy customization?
  • LXC/LXD if You Want: Lightweight VMs requiring less overhead
  • Kubernetes:Multi-node, self-healing workloads, GitOps?
